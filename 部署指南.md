## 🌐 GitHub Pages（仅前端）

**适用场景**：演示前端界面，后端运行在本地或其他服务器

### 步骤 1：配置 GitHub Pages

1. 进入仓库 Settings → Pages
2. Source 选择 **GitHub Actions**
3. 记下你的 GitHub Pages URL：`https://你的用户名.github.io/Sales-Champion-Cloud-Group/`

### 步骤 2：设置环境变量

在仓库 Settings → Actions → Variables 中添加：

- `NEXT_PUBLIC_API_BASE_URL`: 你的后端 API 地址
  - 本地测试：`http://localhost:8000/api/v1`
  - 生产环境：你的公开后端地址

### 步骤 3：运行部署 Action

```bash
# 提交并推送更改
git add .
git commit -m "chore: 添加 GitHub Pages 部署配置"
git push origin main
```

然后在 GitHub 上：
1. 进入 Actions 标签
2. 选择 "Deploy to GitHub Pages"
3. 点击 "Run workflow"

### ⚠️ 注意事项

- GitHub Pages 是静态托管，**后端 API 无法运行**
- 前端需要能访问后端 API（跨域问题需要配置 CORS）
- 本地后端需要公开暴露（可使用 ngrok）

## ⚡ Vercel（全栈部署 - 推荐）

**适用场景**：完整的全栈应用，前后端都需要部署

### 步骤 1：安装 Vercel CLI

```bash
npm install -g vercel
```

### 步骤 2：登录 Vercel

```bash
vercel login
```

### 步骤 3：部署

```bash
# 进入项目目录
cd d:\code\销冠云团\windows\windows\sales-champion-demo-win

# 部署到预览环境
vercel

# 部署到生产环境
vercel --prod
```

### 步骤 4：配置环境变量

在 Vercel 控制台设置：

- `DATABASE_URL`: SQLite 路径或 PostgreSQL 连接字符串
- `JWT_SECRET`: 你的 JWT 密钥
- `REDIS_URL`: Redis 连接字符串（可选）

### ✅ 优势

- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 自动部署（连接 GitHub）
- ✅ 支持 Serverless Functions
- ✅ 免费额度充足

## 🚀 其他部署选项

### 1. Netlify

**适合**：静态站点 + Serverless Functions

```bash
npm install -g netlify-cli
netlify deploy --prod
```

**配置**：
- Build command: `cd frontend && npm run build`
- Publish directory: `frontend/out`

### 2. Railway

**适合**：后端 API 部署

1. 连接 GitHub 仓库
2. 自动检测 Python/FastAPI
3. 设置环境变量
4. 一键部署

### 3. Render

**适合**：全栈应用

- Web Service 部署后端
- Static Site 部署前端
- 免费 PostgreSQL 数据库

### 4. Docker 部署（完整控制）

```bash
# 使用现有的 docker-compose.yml
docker-compose up -d
```

## 🔧 本地后端 + 云端前端（混合部署）

### 使用 ngrok 暴露本地后端

```bash
# 安装 ngrok
choco install ngrok

# 启动后端
python backend/run_server.py

# 暴露本地 8000 端口
ngrok http 8000
```

然后将 ngrok 提供的 URL 设置为前端的 API 地址：

```bash
# 设置环境变量
$env:NEXT_PUBLIC_API_BASE_URL="https://xxx.ngrok.io/api/v1"

# 重新构建前端
cd frontend
npm run build
```
