**⚠️ 重要提示**：由于本项目包含动态路由（如 `/ranking/[staffId]`），**不适合直接部署到 GitHub Pages**。

**推荐使用 Vercel**（全栈部署）或 **Netlify**（静态站点 + Functions）。

## ❌ 为什么不适合 GitHub Pages

GitHub Pages 的限制：

1. ❌ **仅支持静态文件** - 无法运行 FastAPI 后端
2. ❌ **不支持动态路由** - Next.js 的 `[staffId]` 路由需要 SSR 或 ISR
3. ❌ **无服务器端功能** - 无法访问数据库、无法执行 API 调用
4. ❌ **CORS 问题** - 前端需要访问外部 API

**结论**：GitHub Pages 只适合纯静态博客、文档站点，不适合全栈应用。

## ⚡ Vercel（全栈部署 - 推荐）

**适用场景**：完整的全栈应用，前后端都需要部署

### 步骤 1：安装 Vercel CLI

```bash
npm install -g vercel
```

### 步骤 2：登录 Vercel

```bash
vercel login
```

### 步骤 3：部署

```bash
# 进入项目目录
cd d:\code\销冠云团\windows\windows\sales-champion-demo-win

# 部署到预览环境
vercel

# 部署到生产环境
vercel --prod
```

### 步骤 4：配置环境变量

在 Vercel 控制台设置：

- `DATABASE_URL`: SQLite 路径或 PostgreSQL 连接字符串
- `JWT_SECRET`: 你的 JWT 密钥
- `REDIS_URL`: Redis 连接字符串（可选）

### ✅ 优势

- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 自动部署（连接 GitHub）
- ✅ 支持 Serverless Functions
- ✅ 免费额度充足

## 🚀 其他部署选项

### 1. Netlify

**适合**：静态站点 + Serverless Functions

```bash
npm install -g netlify-cli
netlify deploy --prod
```

**配置**：
- Build command: `cd frontend && npm run build`
- Publish directory: `frontend/out`

### 2. Railway

**适合**：后端 API 部署

1. 连接 GitHub 仓库
2. 自动检测 Python/FastAPI
3. 设置环境变量
4. 一键部署

### 3. Render

**适合**：全栈应用

- Web Service 部署后端
- Static Site 部署前端
- 免费 PostgreSQL 数据库

### 4. Docker 部署（完整控制）

```bash
# 使用现有的 docker-compose.yml
docker-compose up -d
```

## 🔧 本地后端 + 云端前端（混合部署）

### 使用 ngrok 暴露本地后端

```bash
# 安装 ngrok
choco install ngrok

# 启动后端
python backend/run_server.py

# 暴露本地 8000 端口
ngrok http 8000
```

然后将 ngrok 提供的 URL 设置为前端的 API 地址：

```bash
# 设置环境变量
$env:NEXT_PUBLIC_API_BASE_URL="https://xxx.ngrok.io/api/v1"

# 重新构建前端
cd frontend
npm run build
```
